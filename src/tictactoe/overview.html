Summary

TicTacToe is an engine for testing tictactoe player strategies, new players need to extend Player. The program serves as a platform, for demonstrating basic principles of Object Oriented Programmer (OOP): Inheritance, Polymorphism and Dynamic Binding. Assignment also serves as a teaching aid to teach someone how to think. Writing an optimal player is not easy. You will have to think about it, test various strategies and sleep on it. It is a good idea to think about how to write your program before going to bed. In the morning have a tablet of paper nearby so you can write any ideas that come to mind while sleeping. Nota Bene TicTacToe your player needs to play with this engine on submit, but you do have to make some changes in Main methods to get program to compile, and you may want to explore various debug modes.

Getting Started, Creating a New Player

  1. Write a basic Random player that extends Player
  2. All extended player classes, must have only a default null constructor, which calls super( realNamewTerm, avatar),
  3. where RealNamewTerm is your real name with current term (such as Joe Student Sp14);
  4. avatar is your combat name.
  5. You have only one default null constructor, no other constructors.
  6. You will need to implement all Abstract methods inside of Player
  7. Modify goForIt.java so that your random player and only your random player interfaces to the engine, initially random player plays against itself
  8. You may need to comment out players, that you don't have
  9. Formulate a strategy to create a better player (i.e., sbusername ), create it, and then test it against your random player
  10. Your submitted player must be your Saddleback Username, both class name and file name must be Saddleback Username
  11. If you have helper classes make sure they are friendly, you can only submit one file i.e., sbusername.java

  • Coordinate (row, col duplet helpers)

  • Board (defines the board MAXROWS x MAXCOLS )

  • SBUsername* extends Player extends VitalStatistics
    It is recommended you write players in this order...
    1. First level Player: pick any square (even one occupied)
    2. Second Level Player: Find an available square (PlayerNext)
    3. Third Level Player: Look for 2 in a row to win Game or block 2 in a row by opponents
    4. Fourth-Final Level: Solve the game, but use previous ones for system testing.

  • MessageTemplate defines fields and autthentication

  • Consists of msgID (integer), details, classPtr

  • Message defines msgID constants, and other methods Specific to game, msgID constants will always be available.

msgID IDNEWGAME first message sent to a player

player must set msg to either IDYES or IDNO, IDYES play another game

msg.details = XCHAR or OCHAR depending upon which token the player has been assigned.

 

then engines sends message ID1..ID9 where ID1 is first move of game; ID2 is second move,

 

ID1 - ID9 will always be sequential; each value incremented by 1. First move will always be 1 (aka Message.ID1)

 

as soon as a winner is found system sends IDWON and IDLOSS to respective players;

on tie after ID9, engines sends IDTIE to both players

TicTacToe

Main engine

To use...

TicTacToe tic = new TicTacToe( );

Players have one of two options for playing a tournament,

Tic.singleUserTestMode( new PlayerName() );

Or

Tic. headToHead( new Player1(), new Player2() );

note you are passing Player pointers to TicTacToe, so one could instantiate a player object and pass it to tic.

Player PlayerXYZ = new PlayerXYZ();

tic.singleUserTestMode( PlayerXYZ );

tic.headToHead( PlayerXYZ, PlayerXYZ );
